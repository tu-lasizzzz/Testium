 Quick Start Guide

                
            

            

                

                    
What is Testium?

                    
Testium is a lightweight tool to test APIs. You can send requests, view responses, and organize your work without any complex setup.

                

                

                    
Sending your first request

                    
1. Select a method (e.g., GET).
2. Enter a URL (e.g., https://dummyjson.com/products).
3. Click Send.

                

                

                    
Using Headers

                    
Switch to the Headers tab to add custom headers (like Content-Type: application/json) if the API requires them.

                

                

                    
Request Body

                    
Use the Body tab for POST or PUT requests. Select JSON and type your data to send information to the server.

                

                

                    
Authentication

                    
If an API needs a token, go to the Auth tab. Choose Bearer Token or API Key and enter your credentials.

                

                

                    
Collections

                    
Click the Save button to store requests in Collections. Use folders to keep them organized in the sidebar.

                

                

                    
Environment Variables

                    
Click Environment in the header to save values (like a base URL). Use them by typing {{VARIABLE_NAME}} in your request.

                

                

                    
Viewing Responses

                    
After sending, the response will appear below. You can see the status code, time, size, headers, and formatted JSON body.

 Free APIs Hub

                
            

            

                
Practice API testing with these free public APIs. Click "Use Example" to try them out.

                
                

                    
                    

                        

                            
FreeAPI

                            Visit FreeAPI 
                        

                        
A free open-source platform with many APIs that beginners can use for learning, frontend practice, backend development, and testing applications. Includes APIs for Random Jokes, Quotes, Users, Products, Weather, Movies, Todos, and Auth.

                        

                            GET
                            https://api.freeapi.app/api/v1/public/randomjokes
                        

                        
                    


                    
                    

                        

                            
DummyJSON

                        

                        
Get mock data for products, recipes, users, and more. Great for frontend testing.

                        

                            GET
                            https://dummyjson.com/products
                        

                        
                    


                    
                    

                        

                            
JSONPlaceholder

                        

                        
Free fake API for testing and prototyping. Provides mock posts, comments, and albums.

                        

                            GET
                            https://jsonplaceholder.typicode.com/posts
                        

                        
                    


                    
                    

                        

                            
ReqRes

                        

                        
A hosted REST-API ready to respond to your AJAX requests. Good for testing auth and users.

                        

                            POST
                            https://reqres.in/api/users

💡 Beginner Tips

                    
    
                        
  • Start with a simple GET request.
    • 
                        
  • Check the response status code after every request.
    • 
                        
  • Use Collections to save requests.
    • 
                        
  • Use Environment Variables instead of writing the same URL multiple times.
    • 
                        
  • Try public APIs before testing your own backend.
